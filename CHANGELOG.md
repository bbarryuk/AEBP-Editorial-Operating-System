### Fixed — Pre-Publish Audit on Week 4 cornerstone post catches a wrong-scenario statute citation and a near-miss anecdote drift, 2026-07-18

Brian asked for a Pre-Publish Audit on post 8342 ("How to Switch Property Managers in California Without Losing a Tenant or Violating Your Lease") and its video companion (8344) before passing them to ChatGPT for final review — the first time this repo's review framework has been run as a deliberate pre-ChatGPT gate-check rather than after the fact. Two real findings, logged in full in `tests/legal/TEST-LEGAL-004.md`:

- **Civil Code §1950.5 was cited for the wrong scenario.** The draft claimed §1950.5 requires a security deposit to "transfer to a successor in interest" when a property owner switches managers. Fetched the actual statute text (via FindLaw's mirror — leginfo.legislature.ca.gov returned empty content on a static fetch, consistent with the known JS-rendering tooling gap already logged for the Richmond rent page in `knowledge/laws/rent-caps.md`) and found §1950.5(i)–(k)'s transfer provisions trigger on termination of the **landlord's interest** — a change of ownership (sale, assignment, death) — not a same-owner change of managing agent. The correct authority for an outgoing property manager's duty to account for and disburse trust funds is Business & Professions Code §10145 (broker trust fund handling). Both posts corrected; `knowledge/laws/change-of-manager-notice.md` created as the first `/knowledge` file on this topic, with primary-source-verified text of Civil Code §1962 (previously cited from a WebSearch summary only) and the corrected §1950.5/§10145 scope distinction.
- **A confirmed AEBP-specific fact drifted into an unconfirmed, more specific elaboration in the same drafting pass.** Brian had confirmed two real facts (owners hold their own security deposits in nearly all cases; AEBP sends a §1962(c) welcome-letter addendum before collecting rent from any new tenant, with a real example letter reviewed). Both were used correctly. But the same pass also added unconfirmed specifics next to them — "we've onboarded from managers who couldn't produce a clean itemized statement," "the single most common gap we find," "a lease that can't be located" — that were never confirmed by Brian. Caught by Claude on a structured self-check against `EVD-ANECDOTE-INTEGRITY` as part of the same review, before Brian read the draft. This is the same failure mode as `TEST-EDITORIAL-001` (post 8006) but with a new variant worth tracking: the invented layer sat directly next to a correctly-used real fact, in the same voice, with nothing distinguishing them — and no one asked for more specificity this time; the drafting process generated that pressure internally.
